# QVAC Assistant — 接口清单

## 基本信息

- **API 版本**: v1
- **通信协议**: HTTP/1.1 (127.0.0.1 only, 零外网连接)
- **后端端口**: 18888
- **Bridge 端口**: 18889 (Electron 内嵌)
- **数据格式**: JSON
- **流式接口**: Server-Sent Events (SSE) / Chunked Transfer

---

## 状态机枚举（前后端统一）

| 状态码 | 名称 | 说明 |
|---|---|---|
| 100 | IDLE | 空闲 |
| 110 | FILE_UPLOADING | 文件上传中 |
| 120 | FILE_PROCESSING | 文件处理中 |
| 130 | EMBEDDING | 向量库构建中（后台轨） |
| 140 | RETRIEVING | RAG 检索中 |
| 150 | LLM_GENERATING | 大模型流式推理中 |
| 160 | ASR_PROCESSING | ASR 转写中（后台轨） |
| 170 | TTS_GENERATING | TTS 流式生成中（后台轨） |
| 175 | TTS_PLAYING | 音频播放中（纯前端态） |
| 180 | EXPORTING | 审计日志导出中 |
| 500 | ERROR | 错误，强制回滚至 100 |
| 999 | NETWORK_LOCKED | 联网锁定（最高优先级） |

## 一、健康检查与状态

### `GET /health`
系统健康检查（无认证）。

**响应示例**:
```json
{
  "status": "ok",
  "models": {
    "llm_loaded": true,
    "embed_loaded": true
  }
}
```

### `GET /api/v1/system/state`
获取当前系统状态快照。

### `GET /api/v1/system/hardware`
获取实时硬件资源信息（GPU 显存、CPU、RAM）。

**响应示例**:
```json
{
  "code": 100,
  "status": "IDLE",
  "data": {
    "gpu_memory_used_mb": 718.1,
    "gpu_memory_total_mb": 6144.0,
    "cpu_utilization_percent": 5.7,
    "ram_used_mb": 9009.1,
    "ram_total_mb": 16070.5
  }
}
```

## 二、对话 API（Chat）

### `POST /api/v1/chat/session`
创建新会话。

**请求体**:
```json
{"api_version": "v1", "title": "新会话"}
```

### `GET /api/v1/chat/sessions`
列出所有会话。

### `GET /api/v1/chat/history?session_id=xxx`
获取会话历史消息。

### `POST /api/v1/chat/send`
发送消息 — 流式 SSE 响应（SSE: text/event-stream）。

**请求体**:
```json
{
  "api_version": "v1",
  "session_id": "sess-xxx",
  "message": "你的问题",
  "enable_rag": true,
  "isolate_mode": "session",
  "context_document_ids": []
}
```

**SSE 流式响应**: 逐 token 推送 `{"token": "..."}`, 结束时 `{"done": true, "full_text": "...", "stats": {...}}`。

### `POST /api/v1/chat/session/close?session_id=xxx`
关闭会话 — 物理销毁 Temp 文档缓存。

### `POST /api/v1/chat/session/rename?session_id=xxx&title=新标题`
重命名会话。

### `POST /api/v1/chat/session/switch?from_session_id=xxx&to_session_id=yyy`
切换会话 — 销毁源会话 Temp 文档。

## 三、知识库 API（Knowledge）

### `POST /api/v1/knowledge/upload`
上传文档（PDF/DOCX/TXT/MD），自动分片、向量化、FAISS 索引。

**表单参数**:
- `file`: 文件（UploadFile）
- `isolate_mode`: global | session | temp
- `session_id`: (可选) session 模式必填
- `folder_id`: (可选) 文件夹 ID

### `GET /api/v1/knowledge/list?session_id=xxx&folder_id=xxx`
列出知识库文件，支持按文件夹和会话筛选。

### `GET /api/v1/knowledge/document/{file_id}/content`
获取文档全文内容（从 rag_chunks 拼接）。

### `DELETE /api/v1/knowledge/delete?file_id=xxx`
三步物理删除（is_deleted=1 → 异步 os.remove → 冷启动自检）。

### `POST /api/v1/knowledge/export`
导出文档为指定格式（txt/md/docx）。

### `POST /api/v1/knowledge/translate`
翻译文档全文（通过 Bridge LLM）。

### `POST /api/v1/knowledge/tts`
文档内容 TTS 语音朗读。

### `GET /api/v1/knowledge/versions/{import_group_id}`
查看文档版本历史。

### `GET /api/v1/knowledge/download/{file_id}`
下载指定版本文档（文件名自动添加时间戳）。

### `PUT /api/v1/knowledge/files/move`
移动文档到文件夹。

### `PUT /api/v1/knowledge/files/isolate`
修改单个文档的隔离模式。

### 文件夹操作
| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/api/v1/knowledge/folders` | 创建文件夹 |
| GET | `/api/v1/knowledge/folders?tree=true` | 列出文件夹（树形） |
| PUT | `/api/v1/knowledge/folders/{folder_id}` | 重命名文件夹 |
| DELETE | `/api/v1/knowledge/folders/{folder_id}` | 删除文件夹 |

### 批量操作
| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/api/v1/knowledge/batch/delete` | 批量删除文档 |
| POST | `/api/v1/knowledge/batch/isolate` | 批量修改隔离模式 |
| POST | `/api/v1/knowledge/batch/translate` | 批量翻译（异步并发） |
| POST | `/api/v1/knowledge/batch/export` | 批量导出打包 ZIP |

## 四、ASR 语音转写 API

### `POST /api/v1/asr/submit`
异步投递 ASR 任务。

**请求体**:
```json
{
  "api_version": "v1",
  "audio_path": "文件本地路径",
  "audio_type": "wav"
}
```

### `POST /api/v1/asr/upload`
ASR 文件上传接口（接受 WAV 文件上传后异步转写）。

### `GET /api/v1/asr/status?task_id=xxx`
轮询 ASR 任务状态（前端每 2s 轮询一次）。

### `GET /api/v1/asr/list`
列出所有 ASR 归档记录。

### `POST /api/v1/asr/import-to-kb`
将 ASR 转写结果一键导入知识库。

### `POST /api/v1/asr/export`
导出 ASR 转写结果为 txt/md/docx。

### 批量操作
| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/api/v1/asr/batch/delete` | 批量删除 ASR 归档 |
| POST | `/api/v1/asr/batch/import-to-kb` | 批量导入知识库 |
| POST | `/api/v1/asr/batch/export` | 批量导出打包 ZIP |

## 五、TTS 语音朗读 API

### `POST /api/v1/tts/stream`
TTS 流式朗读（Chunked Transfer）。

**请求体**:
```json
{
  "api_version": "v1",
  "text": "待朗读文本",
  "voice_model": "male_professional",
  "speed": 1.1
}
```

### `POST /api/v1/tts/abort`
强行销毁 TTS 句柄 — 点击即静音，0 延迟。

**请求体**:
```json
{
  "api_version": "v1",
  "action": "destroy_handler"
}
```

## 六、系统 API（System）

### `POST /api/v1/system/recover`
凭据灾备恢复 — 通过 .key 文件内容或 12 位助记词恢复主密钥。

### `GET /api/v1/system/credential/status`
查询凭据状态。

### `GET /api/v1/system/credential/export`
导出恢复密钥（.key 文件内容），用户必须安全保管。

### `GET /api/v1/system/setup/status`
首次启动引导状态检查。

### `GET /api/v1/system/setup/mnemonic`
生成 12 位中文助记词（灾备恢复备选方案）。

---

## 七、审计日志 API

### `GET /api/v1/log/export?page=1&page_size=20&format=json&api_version=v1`
审计日志分页查询与导出（支持 json/csv 格式）。

### `GET /api/v1/log/export/all?format=json&api_version=v1`
审计日志全量导出（无分页，返回所有记录）。

---

## 日志类型枚举（LogType）

| 类型 | 说明 |
|---|---|
| SYSTEM | 系统事件（启动/关闭/模块初始化） |
| SECURITY | 安全事件（凭据管理/加密操作） |
| SESSION | 会话生命周期（创建/切换/关闭） |
| INFERENCE_SAMPLE | 推理性能采样（tokens/s / TTFT） |
| RAG_RETRIEVAL | RAG 检索事件（命中/未命中） |
| KNOWLEDGE_UPLOAD | 知识库文件上传 |
| KNOWLEDGE_DELETE | 知识库文件删除 |
| ASR_PROCESSING | ASR 任务状态变更 |
| TTS_GENERATING | TTS 生成事件 |
| TTS_ABORT | TTS 销毁事件 |
| EXPORT | 审计日志导出事件 |
| ERROR | 系统异常 |

---

## RAG 检索规范

| 参数 | 值 |
|---|---|
| chunk_size | 512 Token |
| chunk_overlap | 128 Token |
| TopK | 5（默认）/ 10（查找模式） |
| Cosine 相似度阈值 | 0.65 |
| 低相似度返回码 | `code: 145` |
| 隔离模式 | global / session / temp |
