# QVAC Assistant — 项目复现指南

## 1. 项目概述

**QVAC Assistant** 是一款全离线 AI 办公助手，基于 QVAC RAG SDK v0.6.2 构建，支持本地 LLM 对话、RAG 知识库检索增强、ASR 语音转写、TTS 语音朗读等功能。全程无需联网，所有 AI 推理均在本地 GPU 完成。

- **参赛赛道**: 通用设备赛道
- **开源协议**: Apache 2.0
- **GitHub 仓库**: https://github.com/caocname/qvachackstext

## 4. 复现步骤

### 第 1 步：准备模型文件

将 3 个模型文件放入 `data/models/` 目录：

```
data/models/
├── Llama-3.2-1B-Instruct-Q4_0.gguf  (~773 MB)
├── gte-large_fp16.gguf               (~670 MB)
└── ggml-base.bin                     (~148 MB)
```

⚠ **文件名必须完全一致，请勿重命名。**

### 第 2 步：准备 QVAC SDK

将 `@qvac` 文件夹（包含 SDK 及全部子包）复制到 `frontend/node_modules/@qvac/`：

```
frontend/node_modules/@qvac/
├── sdk/
├── llm-llamacpp/
├── embed-llamacpp/
├── transcription-whispercpp/
├── tts-onnx/
├── onnx/
├── rag/
├── decoder-audio/
└── ... (其他子包)
```

### 第 3 步：安装依赖

双击运行 `安装依赖.exe`（或项目根目录 `安装依赖.exe`），此程序将自动：

1. 检查 Node.js v22+ 是否安装
2. 检查 Python 3.12 是否安装
3. 安装前端 npm 依赖（`cd frontend && npm install`）
4. 安装后端 Python 依赖（`pip install -r backend/requirements.txt`）
5. 创建数据目录结构
6. 验证模型文件完整性

### 第 4 步：启动应用

双击运行 `QVAC开发启动器.exe`（或命令行 `.\scripts\dev.ps1`），此程序将：

1. 自动清理端口残留进程
2. 启动 Python 后端服务（FastAPI @ 127.0.0.1:18888）
3. 等待后端 health check 确认就绪
4. 启动 Electron 前端应用（内嵌 Bridge @ 127.0.0.1:18889）
5. 前端关闭后自动终止后端进程

### 第 5 步：验证功能

- 打开前端界面，检查模型加载状态（LLM + Embedding 是否绿标）
- 发送一条对话消息，确认流式推理正常
- 上传一个 PDF 文件到知识库，测试 RAG 检索增强
- 上传一个 WAV 音频，测试 ASR 转写

## 5. 一键部署说明

本项目提供了两个便携式 exe 启动器（PyInstaller 打包，无需系统安装 Python）：

| 文件 | 大小 | 用途 |
|---|---|---|
| `安装依赖.exe` | 7.7 MB | 首次运行，检测环境并安装所有依赖 |
| `QVAC开发启动器.exe` | 8.3 MB | 日常开发/使用一键启动 |

也可通过命令行启动：
```powershell
# 一键全栈启动
.\scripts\dev.ps1

# 仅启动后端
.\scripts\start_backend.ps1

# 仅启动前端
.\scripts\start_frontend.ps1
```

## 6. 架构说明

```
┌─────────────────────────────────────────────────────┐
│                   Electron Frontend                   │
│  ┌──────────┐  ┌──────────────┐  ┌───────────────┐  │
│  │ Renderer  │  │  Preload     │  │  Bridge       │  │
│  │ (HTML/CSS)│  │  (IPC 桥接)  │  │  (:18889)     │  │
│  └──────────┘  └──────────────┘  └───────┬───────┘  │
│                                          │           │
└──────────────────────────────────────────┼───────────┘
                                           │ 127.0.0.1
┌──────────────────────────────────────────┼───────────┐
│                  Python Backend           │           │
│  ┌──────────┐  ┌────────┐  ┌───────────┐│           │
│  │ FastAPI   │  │ RAG    │  │ Kill      ││           │
│  │ (:18888)  │  │ Service│  │ Switch    ││           │
│  └──────────┘  └────────┘  └───────────┘│           │
│                                          │           │
│  ┌──────────┐  ┌────────┐  ┌───────────┐│           │
│  │ LLM      │  │ Embed  │  │ ASR       ││           │
│  │ Service   │  │ Service│  │ Service   ││           │
│  └──────────┘  └────────┘  └───────────┘│           │
│                                          │           │
│  ┌──────────┐  ┌────────┐  ┌───────────┐│           │
│  │ SQLite   │  │ FAISS  │  │ AES-256   ││           │
│  │ (加密)   │  │ Index  │  │ GCM 加密  ││           │
│  └──────────┘  └────────┘  └───────────┘│           │
└──────────────────────────────────────────┘           │
```

- **IPC 通信**: 前后端通过 HTTP (127.0.0.1) 通信，不绑定任何 LAN/WAN 端口
- **Kill Switch**: 独立守护线程每 5s 监控进程网络连接，检测到外联立即锁定
- **数据加密**: SQLite 全表使用 AES-256-GCM 加密，主密钥托管于 Windows 凭据管理器

## 8. 常见问题

**Q: 启动后提示 "后端连接失败"？**
A: 确认 `data/models/` 下有完整的 3 个模型文件，重新运行 `安装依赖.exe`

**Q: 提示 "QVAC SDK 未找到"？**
A: 确认 `frontend/node_modules/@qvac/sdk/package.json` 存在

**Q: 启动后页面黑屏？**
A: 检查 `data/logs/` 下的日志文件，可能是 GPU 内存不足。尝试关闭其他 GPU 应用后重启

**Q: ASR 转写失败？**
A: 确认音频格式为标准无损 WAV（16kHz / 16bit / 单声道），且 `ggml-base.bin` 模型文件存在

**Q: pip install 失败？**
A: 使用 `--break-system-packages` 参数或 `--user` 参数重试
